01장. C언어 기본 구조 및 표준 입출력

C 프로그램의 뼈대와 터미널과 소통하기 위한 기본 입출력 함수를 배웁니다.

  • 기본 프로그램 구조
    • #include <stdio.h>: 표준 입출력 라이브러리(Standard Input/Output)를 포함하는 전처리기 지시문입니다.
    • int main(void): C 프로그램의 실행 진입점(Entry Point)이 되는 메인 함수입니다. 프로그램은 반드시 하나의 main 함수를 가집니다.
    • return 0;: 프로그램이 운영체제에 정상적으로 종료되었음을 알리는 반환 코드입니다.
  • 표준 출력 (printf)
    • printf("문자열"): 지정된 문자열을 화면에 출력합니다.
    • 이스케이프 시퀀스: \n(줄바꿈), \t(탭), \\(백슬래시 출력), \"(큰따옴표 출력) 등을 사용하여 출력 서식을 제어합니다.
  • 표준 입력 (scanf)
    • scanf("서식문자", &변수명): 사용자로부터 입력을 받아 변수에 저장합니다.
    • 대부분의 변수는 주소 연산자(&)를 붙여야 메모리의 올바른 위치에 값을 저장할 수 있습니다.
  • 주석 (Comments)
    • //: 한 줄 주석을 작성할 때 사용합니다.
    • /* 내용 */: 여러 줄에 걸친 블록 주석을 작성할 때 사용합니다.

02장. 변수, 자료형 및 서식 지정자

데이터를 저장하는 메모리 공간의 할당과 각 데이터의 형태를 규정하는 체계입니다.

  • 기본 자료형 (Data Types)
    • 정수형: char(1바이트), short(2바이트), int(4바이트), long(4바이트 이상), long long(8바이트)이 있습니다.
    • 실수형: float(4바이트), double(8바이트, 기본 실수형)이 있습니다.
    • 문자형: char를 사용하며, 내부적으로는 아스키(ASCII) 코드 정수값으로 처리됩니다. 작은따옴표('A')로 표현합니다.
    • sizeof 연산자: sizeof(자료형 또는 변수)를 통해 해당 데이터가 메모리에서 차지하는 바이트 크기를 반환합니다.
  • 서식 지정자 (Format Specifiers)
    • %d: 부호 있는 10진수 정수를 입출력합니다.
    • %f: float형 실수를 입출력합니다 (printf에서는 double도 %f 사용 가능).
    • %lf: scanf에서 double형 실수를 입력받을 때 반드시 사용해야 합니다.
    • %c: 단일 문자를 입출력합니다.
    • %s: 문자열을 입출력합니다.
    • %x, %X: 부호 없는 16진수 정수를 소문자/대문자로 출력합니다.
  • 상수 (Constants)
    • const 키워드를 변수 선언 앞에 붙여 초기화된 값을 이후에 변경할 수 없도록 고정합니다.

03장. 연산자(Operator)와 비트 논리 제어

데이터를 연산하고 조작하기 위한 기호와 하드웨어 제어에 필수적인 비트 연산을 다룹니다.

  • 산술 및 대입 연산자
    • 사칙연산: +, -, *, / (정수끼리의 나눗셈은 소수점 이하를 버림).
    • 나머지 연산자 (%): 나눗셈 후 남은 나머지를 구합니다 (정수형에만 사용 가능).
    • 복합 대입 연산자: a += b는 a = a + b와 동일하게 작동합니다. (-=, *=, /=, %= 포함)
  • 증감 연산자
    • ++a (전위): 값을 1 증가시킨 후 수식에 적용합니다.
    • a++ (후위): 수식에 현재 값을 먼저 적용한 후 1 증가시킵니다. (--도 동일한 원리 적용)
  • 관계 및 논리 연산자
    • 관계 연산자: ==(같음), !=(다름), >, <, >=, <= (참이면 1, 거짓이면 0을 반환).
    • 논리 연산자: &&(AND, 모두 참일 때 참), ||(OR, 하나라도 참이면 참), !(NOT, 참/거짓 반전).
  • 비트 연산자 (Bitwise)
    • 데이터를 이진수 비트 단위로 제어하여 메모리 효율과 연산 속도를 극대화합니다.
    • & (Bitwise AND), | (Bitwise OR), ^ (Bitwise XOR), ~ (Bitwise NOT).
    • << (Left Shift): 비트를 왼쪽으로 밀어냅니다 (2를 곱하는 효과).
    • >> (Right Shift): 비트를 오른쪽으로 밀어냅니다 (2로 나누는 효과).

04장. 조건문 및 반복문 기반 제어 흐름 구조

프로그램의 논리적 실행 순서를 결정하는 분기 처리와 반복 제어 메커니즘입니다.

  • 조건문 (Conditional Statements)
    • if / else if / else 문: 괄호 안의 조건식이 참(0이 아닌 값)일 때 블록 내부를 실행합니다.
    • switch ~ case 문: 정수나 문자식의 결과에 따라 해당하는 case 레이블로 바로 점프합니다.
      • 반드시 break문을 사용하여 해당 블록을 탈출해야 하며, 그렇지 않으면 아래 case 구문이 연속 실행(Fall-through)됩니다.
      • default: 일치하는 case가 없을 때 실행되는 예외 처리 구역입니다.
    • 삼항 조건 연산자: 조건식 ? 참일때의_값 : 거짓일때의_값 형태로 간단한 if-else 구조를 한 줄로 대체합니다.
  • 반복문 (Loop Statements)
    • for 문: for(초기식; 조건식; 증감식) 구조로 반복 횟수가 명확할 때 주로 사용합니다.
    • while 문: 조건식이 참인 동안 계속해서 반복 실행합니다.
    • do ~ while 문: 루프 진입 조건과 무관하게 블록 내부 코드를 무조건 1회는 먼저 실행한 후 조건을 검사합니다.
  • 루프 분기 제어
    • break: 현재 속한 가장 가까운 반복문 블록을 즉시 파괴하고 탈출합니다.
    • continue: 반복문 하단의 남은 코드를 건너뛰고 바로 다음 반복 회차(조건식이나 증감식)로 진행을 강제합니다.

05장. 함수(Function) 정의 및 변수의 스코프

코드의 재사용성을 높이는 모듈화 단위인 함수와 메모리 가시성을 결정하는 변수의 범위입니다.

  • 함수 선언 및 정의
    • 반환형 함수명(매개변수 목록) { 실행문; return 값; } 형태로 구조화합니다.
    • main 함수 상단에 원형(Prototype)을 먼저 선언해야 컴파일러가 함수의 존재를 인식하여 정상 컴파일할 수 있습니다.
    • 반환값이 없으면 반환형을 void로 지정합니다.
  • 매개변수 전달 방식
    • 값에 의한 호출 (Call by Value): 실인자의 '값'만 복사되어 전달되므로 함수 내부에서 매개변수를 변경해도 원본 변수에는 영향을 주지 않습니다.
    • 참조에 의한 호출 (Call by Reference): 포인터를 이용하여 원본 변수의 '주소'를 전달하므로 함수 내부에서 원본 데이터의 직접 수정이 가능합니다 (scanf의 원리).
  • 변수의 유효 범위 (Scope) 및 수명
    • 지역 변수 (Local Variable): 중괄호 {} 블록 내부에서 선언되며 블록이 끝나면 스택(Stack) 메모리에서 자동 소멸됩니다.
    • 전역 변수 (Global Variable): 함수 외부에서 선언되어 프로그램 시작부터 끝까지 데이터(Data) 영역 메모리에 상주하며 모든 함수에서 접근 가능합니다.
    • 정적 변수 (Static Variable): 함수 내부에서 선언되더라도 프로그램 종료 시까지 값이 유지되며, 해당 함수가 여러 번 호출되어도 단 1회만 초기화됩니다.

06장. 1차원 및 다차원 배열(Array) 아키텍처

동일한 자료형의 데이터를 연속된 메모리 공간에 그룹화하여 관리하는 구조입니다.

  • 1차원 배열
    • 자료형 배열명[크기]; 형식으로 선언합니다 (예: int arr[5];).
    • 인덱스는 항상 0부터 시작하여 크기-1까지 접근할 수 있습니다.
    • 초기화: int arr[3] = {1, 2, 3}; 선언과 동시에 중괄호로 값을 대입할 수 있습니다.
    • 배열의 범위를 벗어난 인덱스 참조 시 컴파일러가 오류를 잡아주지 못하며 심각한 런타임 메모리 침범 버그를 유발할 수 있습니다.
  • 다차원 배열
    • int matrix[3][4]; 처럼 행(Row)과 열(Column)을 가지는 2차원 이상의 배열 구조입니다.
    • 메모리 상에는 2차원 구조가 아닌 연속적인 1차원 형태로 길게 펼쳐져(Row-major order) 저장됩니다.
  • 문자 배열과 문자열 (String)
    • C언어는 내장된 문자열 자료형(String)이 없으므로 char 배열을 사용하여 문자열을 구현합니다.
    • 문자열의 끝을 나타내기 위해 항상 널 문자('\0', 아스키값 0)가 마지막 공간에 삽입되어야 하므로 배열 크기는 문자 수보다 1 더 커야 합니다.
    • char str[] = "Hello"; 와 같이 큰따옴표로 초기화 시 널 문자가 자동 포함됩니다.

07장. 포인터(Pointer) 기초 및 메모리 주소 개념

C언어의 핵심이자 메모리 주소를 직접 제어할 수 있게 해주는 강력한 도구입니다.

  • 포인터 변수의 개념
    • 일반 변수가 '데이터'를 저장한다면, 포인터 변수는 데이터가 위치한 '메모리의 시작 주소'를 저장하는 특수 변수입니다.
    • 자료형 *포인터명; 형식으로 선언합니다 (예: int *p;).
  • 주요 포인터 연산자
    • 주소 연산자 (&): 변수 이름 앞에 사용하여 해당 변수가 할당된 실제 메모리 시작 주소를 반환합니다 (예: p = &num;).
    • 간접 참조 연산자 (*): 포인터 변수 앞에 사용하여 그 주소가 가리키는 실제 메모리 공간의 '데이터'에 접근하여 값을 읽거나 수정합니다 (예: *p = 10;).
  • 포인터의 자료형 제약
    • int *p 포인터는 반드시 int형 변수의 주소만 저장해야 합니다. 이는 주소를 읽어올 때 몇 바이트만큼 읽을지(이 경우 4바이트)를 결정하는 잣대가 되기 때문입니다.
    • void 포인터 (void *): 가리키는 자료형이 정해지지 않은 범용 포인터로, 어떤 타입의 주소든 담을 수 있으나 역참조(*)나 포인터 연산을 수행하려면 반드시 형 변환(Casting)이 필요합니다.

08장. 포인터와 배열의 관계 및 문자열 제어

배열 이름의 숨겨진 포인터 특성과 이를 응용한 문자열 함수 라이브러리를 학습합니다.

  • 배열 이름의 정체
    • 배열의 이름은 그 배열의 첫 번째 원소(인덱스 0)를 가리키는 상수 포인터와 동일하게 취급됩니다.
    • 즉, arr과 &arr[0]은 완전히 동일한 주소값을 가집니다.
  • 포인터 연산 (Pointer Arithmetic)
    • 포인터에 1을 더하면 단순 정수 1이 더해지는 것이 아니라, 가리키는 '자료형의 바이트 크기'만큼 주소가 이동합니다.
    • int *p = arr; 일 때, *(p + 1)은 arr[1]의 값을 참조합니다.
  • 문자열 제어 라이브러리 (<string.h>)
    • strlen(str): 널 문자를 제외한 문자열의 순수 길이를 반환합니다.
    • strcpy(dest, src): src의 문자열을 dest 배열로 널 문자를 포함하여 복사합니다 (할당 연산자 = 로는 배열 복사가 불가능함).
    • strcat(dest, src): dest 문자열 끝에 src 문자열을 이어 붙입니다 (병합).
    • strcmp(str1, str2): 두 문자열을 사전식으로 비교하여 같으면 0, str1이 크면 양수, 작으면 음수를 반환합니다.

09장. 구조체(Struct), 공용체 및 사용자 정의 자료형

서로 다른 자료형의 변수들을 하나의 단위로 묶어 객체 지향적 접근의 기초를 제공합니다.

  • 구조체 (struct) 정의
    • struct 키워드를 사용하여 다양한 자료형의 멤버 변수들을 하나의 새로운 자료형으로 캡슐화합니다.
    • 점(.) 연산자를 사용하여 구조체 변수 내부의 개별 멤버에 접근합니다 (예: student.age = 20;).
  • 구조체 포인터와 화살표(->) 연산자
    • 포인터로 구조체를 다룰 때는 (*p).age 처럼 괄호와 점 연산자를 쓰거나, 더 직관적인 화살표 연산자를 사용하여 p->age 형태로 멤버에 접근합니다.
  • typedef를 이용한 타입 재정의
    • typedef struct Student { ... } StudentType; 처럼 선언하여, 매번 struct 키워드를 쓰는 번거로움을 줄이고 자료형 이름을 간략화합니다.
  • 공용체 (union)
    • 구조체와 선언 방식은 같으나, 모든 멤버 변수들이 가장 큰 멤버의 메모리 공간을 공유합니다. 한 번에 하나의 멤버만 온전한 값을 유지할 수 있어 하드웨어 제어나 메모리 절약 시 사용됩니다.

10장. 동적 메모리 할당(Dynamic Allocation) 및 해제

컴파일 시점이 아닌 런타임(실행 중)에 필요한 만큼의 힙(Heap) 메모리를 탄력적으로 빌려오는 기법입니다.

  • 동적 할당 라이브러리 (<stdlib.h>)
    • malloc(size): 전달된 바이트(size) 크기만큼 메모리를 할당하고 첫 번째 주소를 void* 형태로 반환합니다. 할당된 메모리는 쓰레기값으로 초기화되어 있습니다.
    • calloc(count, size): count 개의 원소를 가지는 배열 크기만큼 메모리를 할당하며, 내부를 전부 0으로 자동 초기화합니다.
    • realloc(ptr, new_size): 이미 동적 할당된 메모리(ptr)의 크기를 new_size로 재조정하여 확장하거나 축소합니다.
  • 메모리 누수(Memory Leak) 방지와 free()
    • 지역 변수와 달리 힙 메모리에 할당된 공간은 프로그램이 자동으로 회수하지 않습니다.
    • 사용이 끝난 후 반드시 free(ptr); 함수를 호출하여 메모리를 운영체제에 반납해야 시스템 다운을 막을 수 있습니다.
  • 할당 실패 처리 검증
    • 운영체제에 남은 램(RAM) 공간이 부족하면 malloc은 NULL 포인터를 반환합니다. 따라서 할당 직후 반드시 if (ptr == NULL) 검사 분기를 작성하여 크래시를 방지해야 합니다.

11장. 파일 입출력(File I/O) 및 스트림 제어

휘발성인 램(RAM)의 데이터를 영구 보존하기 위해 하드디스크 외부 파일로 저장하고 읽어오는 기술입니다.

  • 파일 스트림 열기와 닫기
    • fopen("파일명", "모드"): 파일을 열고 FILE 구조체 포인터를 반환합니다. 실패 시 NULL을 반환합니다.
    • 파일 접근 모드: "r" (읽기), "w" (쓰기, 기존 내용 삭제됨), "a" (추가 쓰기). 뒤에 "b"를 붙이면 바이너리 모드("rb", "wb")로 동작합니다.
    • fclose(fp): 사용이 끝난 파일 포인터를 닫아 파일 점유 잠금을 해제하고 버퍼 데이터를 안전하게 디스크에 기록합니다.
  • 텍스트 파일 입출력 함수
    • fprintf(fp, "서식", 데이터): 지정된 파일(fp) 내부에 printf와 동일한 서식 형태로 데이터를 기록합니다.
    • fscanf(fp, "서식", &변수): 파일로부터 데이터를 읽어들여 변수에 매핑합니다. 파일의 끝(EOF)에 도달하면 EOF 값을 반환합니다.
    • fgets(str, size, fp): 파일에서 한 줄(엔터 키 기준)씩 문자열 형태로 안전하게 통째로 읽어옵니다.

12장. 전처리기(Preprocessor), 매크로 및 분할 컴파일

소스 코드가 컴파일러로 넘어가기 직전에 먼저 수행되는 텍스트 치환 작업과 대형 프로젝트 관리법입니다.

  • 매크로 정의 (#define)
    • #define PI 3.141592: 소스 코드 내부의 PI 기호를 모두 3.141592로 치환하는 기호 상수를 생성합니다 (메모리를 차지하지 않습니다).
    • #define MAX(a, b) ((a) > (b) ? (a) : (b)): 인자를 받는 매크로 함수를 정의합니다. 연산자 우선순위 오류를 막기 위해 반드시 매개변수마다 괄호로 철저히 감싸야 합니다.
  • 헤더 파일 포함 (#include)
    • <stdio.h>: 꺾쇠 괄호는 컴파일러가 설치된 표준 시스템 디렉토리에서 라이브러리를 탐색합니다.
    • "myheader.h": 큰따옴표는 현재 개발자가 작업 중인 프로젝트 로컬 폴더에서 헤더 파일을 우선 탐색합니다.
  • 조건부 컴파일 (#ifdef, #ifndef)
    • 특정 매크로가 정의되어 있는지 여부에 따라 컴파일에 포함할 코드 블록을 활성화하거나 비활성화합니다.
    • 헤더 파일 중복 포함 방지(Include Guard)를 위해 #ifndef HEADER_H, #define HEADER_H ... #endif 패턴이 필수적으로 사용됩니다.
  • 분할 컴파일 기초
    • 소스 코드가 길어지면 여러 개의 .c 파일과 .h 헤더 파일로 나누어 작성한 후, 컴파일 단계(Object 파일 생성)와 링킹 단계(Linker)를 거쳐 하나의 실행 파일(.exe)로 결합 빌드합니다.
    • extern 키워드를 사용하여 다른 소스 파일에 선언된 전역 변수를 공유하여 참조할 수 있습니다.
