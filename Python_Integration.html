01장. 변수, 자료형 개요 및 주석 리터럴

파이썬 데이터의 기본적인 분류 체계와 주석 처리 방식을 배웁니다.

  • 변수
    • 리터럴이 저장되는 공간입니다.
  • 자료형
    • 숫자형
      • int 정수입니다.
      • float 실수입니다.
      • complex 복소수입니다.
    • 논리형
      • bool 참과 거짓을 표현합니다.
    • 시퀀스형
      • str 문자열입니다.
      • list 객체의 순서 있는 집합입니다.
      • tuple 객체의 순서 있는 집합입니다.
    • 매핑형
      • dict 키와 값으로 구성됩니다.
    • 집합형
      • set 중복이 불가능한 항목들의 모임입니다.
      • frozenset입니다.
  • 리터럴 (Literals)
    • 일상 생활에서 사용되며 코드에 직접 사용할 수 있는 변수 저장 전의 '있는 그대로의 값'입니다.
  • 주석 (Comment)
    • 프로그램 실행 도중 인터프리터가 무시하며 가독성을 높이는 한 줄 주석(#) 및 여러 줄 주석(""" 또는 ''')입니다.
  • 식별자 명명 규칙 및 키워드
    • 영어 대소문자, 숫자, 밑줄(_) 조합이 가능하며 숫자가 맨 앞에 올 수 없고 예약어는 변수명으로 사용이 불가합니다.
    • 키워드 목록: False, None, True, and, as, assert, async, await, break, class, continue, def, del, elif, else, except, finally, for, from, global, if, import, in, is, lambda, nonlocal, not, or, pass, raise, return, try, while, with, yield가 있습니다.
  • id() 함수
    • 객체의 고유 식별 값인 '객체 식별자'를 반환하며, 불변성 객체와 가변성 객체의 수정 전후 주소 변화 판별에 활용합니다.

02장. 표준 입출력 제어, 상세 서식 및 내장 math 모듈

프로그램의 외부 소통 스트림인 입력과 출력 포맷팅, 그리고 기본 수학 연산 패키지를 정의합니다.

  • 입출력
    • input()
      • 키보드 입력을 무조건 문자열(str) 형태로 반환합니다.
    • 형 변환
      • int(): 입력된 데이터를 정수형으로 변환합니다.
      • float(): 입력된 데이터를 실수형으로 변환합니다.
  • 문자열 포맷팅 시스템
    • f-string: 문자열 앞에 f 접두어를 붙이고 중괄호 {} 안에 변수를 직접 넣어 변수의 값을 삽입하는 간편한 방식입니다.
    • % 포맷 방식: 문자 내부 % 기호를 사용하여 대입 처리합니다.
      • %s: 문자열 매핑입니다.
      • %d: 정수형 매핑입니다.
      • %f: 실수형 매핑입니다.
    • str.format(): 중괄호 {} 정수 인덱스나 키워드 인자를 사용하여 데이터를 포맷팅 삽입하는 메서드 방식입니다.
  • 상세 형식 지정자
    • :10f → 전체 자릿수 10자리 공간 확보 후 실수를 표현합니다.
    • :.2f → 소수점 이하 둘째 자리까지 반올림하여 표현합니다.
  • math 모듈
    • 수학 계산 함수와 상수를 정의해 놓은 모듈로 import math를 통해 활용합니다.
    • math.pi: 원주율 상수입니다.
    • math.e: 자연로그의 밑인 오일러 상수입니다.

03장. 함수 정의 규격, 내장 함수 가이드 및 익명 람다

비즈니스 로직의 단위화인 함수의 선언 메커니즘과 파이썬 기본 제공 내장 함수 API 가이드입니다.

  • 함수 정의
    • def function_name(parameters): 형태의 함수 머리와 들여쓰기 된 함수 몸체로 구성됩니다.
    • arguments: 함수를 호출할 때 실제로 전달하는 데이터 값입니다.
    • return: 함수의 동작 결과를 반환하며 생략 시 None 객체를 리턴합니다.
  • 람다 함수
    • lambda [parameters] : 연산식 → 이름이 없고 return 키워드 없이 한 줄 연산식을 반환하는 익명 함수입니다.
  • 내장 함수 종합 레퍼런스
    • abs(): 숫자의 절댓값을 반환합니다.
    • pow(x, y): x의 y 거듭제곱을 반환합니다.
    • round(x): 실수 x에 가장 가까운 정수를 반환합니다.
      • round(x, y): 실수 x에서 소수점 아래 y+1위에서 반올림하여 소수점 아래 y자리까지 표현합니다.
    • divmod(a, b): 나누기 몫과 나머지로 구성된 한 쌍의 튜플을 반환합니다.
    • min(): 인자들 또는 반복 가능 객체 내부의 최솟값을 반환합니다.
    • max(): 인자들 또는 반복 가능 객체 내부의 최댓값을 반환합니다.
    • sum(): 반복 가능 객체 내부 숫자들의 총합을 계산합니다.
    • bin(): 숫자를 0b가 선행하는 2진수 문자열로 변환합니다.
    • oct(): 숫자를 0o가 선행하는 8진수 문자열로 변환합니다.
    • hex(): 숫자를 0x가 선행하는 16진수 문자열로 변환합니다.
    • format(): 숫자 또는 문자열을 지정된 형식으로 변환합니다.
      • format(a, 'b'): 2진수 문자열로 변환합니다.
      • format(a, 'o'): 8진수 문자열로 변환합니다.
      • format(a, 'x'): 16진수 문자열로 변환합니다.
      • format(a, ','): 세 자리마다 천 단위 쉼표(콤마)를 삽입합니다.
      • format(a, 'E'): 지수 표기법 문자열로 전환합니다.
      • format(a, '%'): 백분율 서식 적용 문자열로 전환합니다.
    • sorted(): 객체의 항목을 정렬한 새로운 오름차순 리스트를 반환합니다.
      • reverse=True 인자를 사용하면 내림차순 정렬 처리합니다.
      • key= 키워드 인자에 함수나 람다식을 지정하여 커스텀 정렬 기준 수립이 가능합니다.
    • reversed(): 시퀀스형 데이터 객체의 순서를 역순으로 뒤집은 이터레이터를 반환합니다.
    • all(): 반복 가능 객체의 모든 요소가 참(True)이거나 비어있으면 True를 판별하여 반환합니다.
    • any(): 하나라도 True인 요소가 있으면 True를 판별하여 반환합니다 (비어있으면 False입니다).
    • help(): 대화형 모드에서 내장 함수나 객체에 대한 상세 정보 도움말 문서를 출력합니다.

04장. 제어 흐름 분기 구조 (조건문과 반복문 매트릭스)

논리 조건에 의거하여 흐름을 쪼개는 조건문과 횟수 또는 상태 기반의 루프 제어 처리를 명세합니다.

  • 조건문
    • if 문
      • 프로그램 실행 순서를 조건에 따라 변경할 수 있으며, 특정 조건이 참일 때만 실행되는 코드 블록을 정의합니다.
    • if else 문
      • else: 블록을 배치하여 if 조건이 만족되지 않을 경우 실행될 대체 코드를 지정합니다.
    • if elif else 문
      • 여러 조건이 계속 연결되면서 해당 조건이 True이면 바로 하단 들여쓰기를 실행하고 제어 구조를 종료합니다.
    • 조건 연산자 (Ternary Operator)
      • exp1 if condition else exp2 → 한 줄 구조 조건 제어 연산식입니다.
  • 반복문
    • for in 문
      • 반복 횟수가 명확히 지정되어 지정 객체 루프를 완료하면 종료되는 모음 기반 구조입니다.
      • 반복 가능 객체 (Iterable): list, tuple, str, range, set, dict가 있습니다.
      • 자료형 range 매개변수 구조
        • range(n): 0에서 시작하여 n 이전까지 정수 시퀀스를 생성합니다 (차이 1).
        • range(s, n): s에서 시작하여 n 이전까지 정수 시퀀스를 생성합니다 (차이 1).
        • range(s, n, t): s에서 시작하여 n 이전까지 t의 간격으로 정수 시퀀스를 생성합니다.
    • while 문
      • 명시적으로 반복 횟수를 지정하지 않고 조건(condition)이 True인 동안 반복 실행되며, 조건이 False가 되면 종료됩니다.
    • 루프 반복 제어어
      • break 문: else: 블록을 가동하지 않고 반복 상위 루프 구조를 무조건 즉시 파괴 및 종료합니다.
      • continue 문: 루프 하단 잔여 몸체 문장을 실행하지 않고 다음 회차 반복 단계 헤더로 강제 진행합니다.

05장. 문자열 제어 기술, 이스케이프 및 str 내장 메서드군

파이썬의 핵심 텍스트 스트림 자료형인 str의 정밀 가공 기술 및 내장 편의 메서드 체계입니다.

  • 문자열 기본 내장 함수
    • str(obj): 지정 객체를 문자열 표현으로 캐스팅하여 반환합니다.
    • len(문자열): 대상 텍스트 스트림의 전체 문자 개수 길이를 반환합니다.
    • ord(c): 단일 문자 c의 고유 유니코드 정수 값을 반환합니다.
    • chr(n): 정수 코드 값 n에 대응하는 유니코드 단일 문자를 반환합니다.
  • 이스케이프 시퀀스 (Escape Sequence)
    • \\ : 역슬래시(\) 문자 표현입니다.
    • \' : 작은따옴표 문자 표현입니다.
    • \" : 큰따옴표 문자 표현입니다.
    • \a : 벨소리 경보 알람 기능입니다.
    • \b : 백스페이스 문자 제어입니다.
    • \n : 새 줄 줄바꿈 제어입니다.
    • \r : 캐리지 리턴 (같은 줄 맨 앞으로 포인터 이동)입니다.
    • \t : 수평 탭 공백 제어입니다.
    • \v : 수직 탭 제어입니다.
  • 문자열 API 전용 메서드
    • '문자열'.split(): 공백문자 블록(스페이스, 탭, 줄바꿈 등)을 기본 기준으로 문자열을 쪼개어 리스트로 변환합니다.
    • '문자열'.rsplit(): 우측에서부터 분리 수행을 처리하며 maxsplit 인자로 횟수 제한이 가능합니다.
    • '문자열'.join(): 인자로 전달된 문자열 리스트 요소 항목들 사이에 기준 '문자열'을 결합하여 삽입하고 반환합니다.
    • '문자열'.replace(a, b): 내부 전체 문자열 구조에서 소스 패턴 a를 타겟 패턴 b로 완전 일괄 치환합니다.
    • '문자열'.zfill(폭): 지정한 폭 크기 공간에서 문자열을 우측 정렬 후 빈 왼쪽 앞 영역을 0으로 채웁니다.
    • '전체문자열'.count('부분문자열'): 타겟 텍스트 내부에서 부분 문자열 패턴의 전체 출현 횟수를 카운트합니다.
    • '전체문자열'.find(sub): 부분 패러다임 sub이 시작되는 첫 인덱스를 탐색하여 반환합니다 (데이터 부재 시 -1을 안전하게 리턴합니다).
    • '전체문자열'.index(sub): find()와 동일하게 작동하나 매칭 데이터 부재 시 ValueError 예외 크래시가 발생합니다.
    • '문자열'.upper(): 대상 문자를 전부 대문자로 변환 처리합니다.
    • '문자열'.lower() / .casefold(): 대상 문자를 소문자로 일괄 변환합니다 (casefold는 강력한 대소문자 무시 비교용입니다).
    • '문자열'.swapcase(): 대문자는 소문자로, 소문자는 대문자로 반전시켜 교체하고 반환합니다.
    • '문자열'.title(): 단어의 첫 글자만 대문자로 변환합니다.
    • '문자열'.center(폭, '문자'): 지정 폭 정렬 영역 중앙에 문자를 배치하고 좌우 공백을 지정 문자로 보충합니다.
    • '문자열'.ljust(폭, '문자'): 지정 폭 내 좌측 정렬 후 빈 우측 공간을 채웁니다.
    • '문자열'.rjust(폭, '문자'): 지정 폭 내 우측 정렬 후 빈 좌측 공간을 채웁니다.
    • '문자열'.strip() / .lstrip() / .rstrip(): 양쪽 / 왼쪽 / 오른쪽 영역의 불필요 공백 및 지정 문자를 제거합니다.

06장. 리스트·튜플 구조, 시퀀스 슬라이싱 및 바이트 인코딩

순서가 보장되는 선형 자료 구조 리스트와 불변 시퀀스 튜플, 메모리 복사 및 인코딩 원론입니다.

  • 리스트(List) 조작 메서드
    • list(iterable): 인자 반복 가능 구조를 유연한 리스트 형태로 캐스팅 가동합니다.
    • insert(index, item): 타겟 첨자 index 위치를 지정하여 해당 위치 바로 앞자리에 객체를 삽입합니다.
    • append(): 리스트 구조의 최하단 종단 위치에 인자 항목을 무조건 추가합니다.
    • extend(iterable): 다른 반복 가능 객체의 내부 항목들만을 추출하여 기존 리스트 볼륨을 확장합니다.
    • remove(obj): 동일 객체 obj를 탐색하여 첫 항목을 제거합니다 (데이터 부재 시 Error가 발생합니다).
    • pop(index): index 위치 항목을 리스트에서 완전히 꺼내어 반환합니다 (인자 생략 시 맨 마지막 요소가 자동 탈출합니다).
    • del 문: 뒤에 연이어 기술한 특정 인덱스 항목이나 변수 자체를 메모리에서 완전 삭제합니다.
  • 인덱싱 및 슬라이싱 아키텍처
    • 시퀀스 개요: 위치 기반 정렬 항목 모임으로 str, list, tuple, range, bytes, bytearray 등이 포함됩니다.
    • 인덱싱: 0부터 시작하는 양수 첨자와 -1부터 시작하는 음수 첨자를 대괄호 내부에 명시하여 단일 참조합니다.
    • 첨자 범위 제한: 0부터 [len-1] 또는 -1부터 [-len]까지 유효하며 초과 시 IndexError가 발생합니다.
    • 가변/불변 격리: 리스트는 첨자 대입 수정이 가능하나 문자열 및 튜플은 생성 후 인덱싱 대입 수정이 불가능합니다.
  • 시퀀스 공통 가동 메서드
    • count(obj): 시퀀스 컨테이너 내에 타겟 객체 obj가 존재하는 전체 중복 빈도수를 리턴합니다.
    • index(obj): 특정 객체 obj가 위치한 첫 번째 고유 첨자 위치를 반환합니다.
  • 첨자를 사용한 슬라이싱 (Slicing)
    • sequence_obj[start:stop:step] 규격 매커니즘이 작동합니다.
    • start: 추출을 시작할 시작 인덱스 포인트입니다 (기본값: 0).
    • stop: 추출을 마칠 종단 인덱스로 stop 위치의 실제 항목 데이터는 포함되지 않습니다 (기본값: 끝).
    • step: 슬라이싱 간격 가중치이며 음수 값을 대입 시 역순 방향 슬라이싱 빌드가 가동됩니다.
    • 슬라이스 대입: 리스트 자료형의 경우 슬라이싱 영역 지정 후 대입 연산자 왼쪽에 배치하여 구역을 일괄 수정할 수 있습니다.
  • 튜플 (Tuple)
    • 표현 방법: 소괄호 기호 ( )를 사용하여 선언하며 소괄호 자체는 생략이 가능합니다.
    • 단일 항목 선언 주의점: 원소가 하나뿐인 튜플을 선언 시에는 타입 오해가 없도록 반드시 뒤에 콤마(,) 기입이 필수입니다 (예: t = 1, 혹은 (1,)).
    • 성질: 리스트와 완벽히 동일한 시퀀스이나 생성 직후 삽입, 삭제, 항목 내용 수정이 완전히 봉쇄된 불변(Immutable) 객체입니다.
    • 튜플을 채택하는 이유
      • 속도: 내부 최적화가 실행되어 가변 리스트 대비 처리 속도가 효율적입니다.
      • 보안: 실수나 외부 오염으로 인한 데이터 변형을 원천 차단합니다.
      • 사전 키 연동: 가변 객체는 불가능한 딕셔너리의 고유 Key 역할을 수행할 수 있습니다 (Hashable 프로토콜을 충족합니다).
    • 패킹(Packing)과 언패킹(Unpacking)
      • 튜플 패킹: 복수의 독립 데이터 자원들을 단일 튜플 구조 객체로 묶어 변수에 저장하는 행위입니다.
      • 튜플 언패킹: 패킹된 튜플 컨테이너의 내부 개수와 완벽히 일치하는 개수의 변수들에 데이터를 동시 대입하여 분할합니다.
  • 인코딩/디코딩 바이트 메서드
    • 인코딩: bytes(text, 'utf-8') 인터페이스 연동으로 텍스트 스트림을 시스템 통신용 이진 바이트 객체로 변환합니다.
    • 디코딩: binary_data.decode('utf-8') 시스템 처리를 거쳐 전송받은 바이트 스트림을 인간이 판독 가능한 str로 환원합니다.

07장. 매핑형 딕셔너리 구조, 안전 조회 프로토콜 및 세트 집합 연산

정수 인덱스가 아닌 고유 키(Key) 매핑 기반 사전 구조와 순서가 없는 중복 차단 집합 세트의 상세 명세입니다.

  • 딕셔너리 (Dictionary)
    • 개요: 중괄호 내부에 {Key: Value} 쌍의 구조를 항목으로 유지하는 매핑 자료형입니다.
    • Key 제한성: 중복된 키는 내부에 공존할 수 없으며 가변 객체(list, set, dict)는 딕셔너리의 키로 선언이 불가합니다 (불변성이 필수입니다).
    • 딕셔너리 고유 편의 내장 메서드
      • .keys(): 사전에 등록된 고유 키(Key) 목록들만을 추출하여 dict_keys 반복 가능 객체를 반환합니다.
      • .values(): 사전 내부의 데이터 값(Value) 목록들만을 모아 dict_values 객체를 반환합니다.
      • .items(): 키와 값을 하나의 튜플 쌍으로 묶어낸 (Key, Value) 구조 목록을 반환합니다.
      • .get(key, default): 키가 사전에 없어도 다운되지 않고 None 또는 지정 커스텀 기본값을 리턴하는 에러 방어형 메서드입니다.
      • .setdefault(key, default): 타겟 키가 부재할 시 지정 기본값을 사전에 영구 삽입 후 해당 값을 반환합니다.
      • .pop(key): 지정 키 항목을 삭제 처리하며 그 삭제되는 값을 외부로 리턴하여 반환합니다.
      • .popitem(): 딕셔너리에 가장 마지막에 입력 처리된 (Key, Value) 항목 튜플을 꺼내고 삭제합니다.
      • .clear(): 기존 사전 내부의 모든 요소를 완전 파괴하여 빈 딕셔너리 상태로 초기화합니다.
      • .update(other_dict): 다른 사전을 병합 처리하며 동일 키가 충돌할 시 인자로 전달된 값으로 덮어씁니다.
  • 세트 (Set)
    • 개요: 중복을 철저히 불허하며 순서 개념이 없는 고유 수학적 원소의 모임 구조입니다.
    • frozenset: 일반 세트와 동일한 연산 특성을 지니나 생성 직후 내부 요소의 추가/삭제가 완벽히 불가능한 불변형 집합입니다.
    • 주요 원소 가동 메서드: .add(원소)(항목 추가), .remove(원소)(부재 시 에러 발생 삭제), .discard(원소)(에러 없이 안전 삭제)가 있습니다.
    • 집합 수학 연산자 매트릭스
      • A | B (.union): 두 집합의 요소를 전부 결합하는 합집합입니다.
      • A & B (.intersection): 공통 원소만 추려내는 교집합입니다.
      • A - B (.difference): 집합 A에만 존재하는 독점 원소를 남기는 차집합입니다.
      • A ^ B (.symmetric_difference): 대칭 차집합 (교집합 영역을 제외한 나머지 여집합의 모임)입니다.

08장. 내장 루프 결합 함수(zip, enumerate) 및 전역/지역 변수 범위

복수의 컬렉션을 효율적으로 병렬 제어하는 기법과 메모리 상의 격리 범위 네임스페이스를 상세 서술합니다.

  • zip(*iterables, strict=False) 매개변수 구조 분석
    • 기본 동작: 동일한 인덱스 위치에 배치된 복수 객체의 원소들을 묶어 튜플 구조의 반복자(Iterator) 객체로 리턴합니다.
    • 반환 타입: 자료형 <class 'zip'> 객체를 반환하며, 내부 데이터를 확인하거나 재사용하려면 list() 또는 tuple()로 변환할 수 있습니다.
    • strict=False (기본값): 인자로 전달된 여러 이터레블 객체들의 '길이가 서로 다를 때', 가장 짧은 객체의 길이에 맞추어 연산을 자동 축소 마감하고 남은 긴 데이터는 안전하게 버림 처리합니다.
    • strict=True (최신 파이썬 규격): 전달된 복수 컬렉션 객체들의 길이가 단 1개라도 불일치할 경우, 데이터를 임의로 버리지 않고 즉시 ValueError 예외 오류 크래시를 발생시켜 정밀 데이터 검증 프로세스를 지원합니다.
  • enumerate(iterable, start=0) 매개변수 구조 분석
    • 기본 동작: 루프 회차 카운팅 변수를 개발자가 수동 제어하지 않고, 시퀀스의 자동 정수 일련번호와 실제 데이터 값을 (index, value) 쌍의 튜플로 안전하게 동시 추출합니다.
    • start=정수: 인덱스 번호가 무조건 0부터 시작하는 제약을 해제하고, 지정한 '정수' 값부터 번호가 순차 증가하도록 가중치 오프셋을 설정합니다 (예: start=1 선언 시 1번부터 인덱싱 부여).
  • 네임스페이스 가시성 범위: 변수 스코프 (Scope) 원론
    • 지역 변수 (Local Variable): 함수 블록 내부(def)에서 대입 연산자(=)를 통해 생성된 변수로, 함수 실행이 끝나는 순간 즉시 메모리에서 완전 파괴 및 소멸합니다.
    • 전역 변수 (Global Variable): 함수 외부 최상위 루트 영역에 적재되어 프로그램이 구동을 마칠 때까지 활성화 상태를 유지하며 전역 영역에서 상시 참조 조회가 가능합니다.
    • global 키워드 선언 규칙: 함수 내부 로직에서 함수 바깥에 있는 전역 변수 데이터를 직접 덮어쓰고 수정(Write)하고자 할 때 매개변수 영역 아래에 global 전역변수명을 명시적으로 확정 선언해 주어야 합니다.

09장. 가변 인자(*args, **kwargs) 매킹 및 특수 매개변수 처리 아키텍처

함수 호출부에서 전달하는 인자의 개수와 타입을 정형화하지 않고 기호 구분자를 통해 인자 전달 방식을 엄격히 통제하는 특수 파라미터 매트릭스입니다.

  • 가변 위치 인자 (*args) 매커니즘 명세
    • 정의 및 구조: 함수 정의문 헤더 매개변수 이름 앞에 단일 별표 * 기호를 수식하여 가동합니다.
    • 패킹(Packing) 프로토콜: 호출부에서 전달하는 개수가 고정되지 않은 무수한 위치 기반 인자(Positional Arguments)들을 인터프리터가 실시간 수집하여 단일 튜플(Tuple) 객체 하나로 묶어 함수 몸체 내부로 안전하게 인입합니다.
    • 언패킹(Unpacking) 활용: 호출 시 리스트나 튜플 객체 식별자 이름 바로 앞에 *를 붙여 함수 인자로 던지면, 내부의 원소들을 낱개의 개별 위치 인자로 해체 전개하여 매핑 전달을 수행합니다.
  • 가변 키워드 인자 (**kwargs) 매커니즘 명세
    • 정의 및 구조: 함수 정의문 매개변수 이름 앞에 이중 별표 ** 기호를 명시하여 가동합니다.
    • 딕셔너리 패킹: 호출부에서 명시한 key_name = value_content 형태의 모든 이름 지정 키워드 인자들을 남김없이 포획하여 단일 딕셔너리(Dictionary) 구조 객체로 패킹 빌드한 뒤 내부로 격리 인입합니다.
  • 기호 기법 슬래시(/)와 별표(*)를 사용한 특수 매개변수 통제 메커니즘
    • 표준 선언 서식 규격: def func([pos], /, [pos_or_kwds], *, [kwds])입니다.
    • 기호 슬래시 (/) 의 규칙: 위치 전용(Position-only) 인자의 경계를 정의함. 슬래시(/) 기호보다 왼쪽에 명시된 매개변수들은 함수 호출 시 무조건 '위치 인자'로만 전달해야 하며, 키워드 인자 형태(pos1=10)로 이름을 지정하여 전달할 경우 즉시 TypeError: positional-only argument... 에러가 발생합니다.
    • 기호 별표 (*) 의 규칙: 키워드 전용(Keyword-only) 인자의 경계를 정의함. 별표(*) 기호보다 오른쪽에 배치된 매개변수들은 함수 호출 시 반드시 '키워드 인자' 형태로 이름을 명시하여 값을 대입해야 하며, 단순 위치 인자로 값을 넘기려고 시도하면 TypeError: takes X positional arguments... 에러가 발생합니다.
    • 기호가 없는 일반적인 환경: 함수 헤더에 /와 * 기호가 명시되지 않은 경우에는 인자 전달 방식에 제한이 없으므로 위치 인자와 키워드 인자 방식을 개발자가 자유롭게 혼용 대입이 가능합니다.
  • 함수 파라미터 배치 선언 규칙의 절대성
    • 여러 형태의 인자를 단일 함수 헤더에 혼합하여 선언할 경우, 반드시 아래의 순서 매트릭스를 엄격하게 준수해야 하며 위반 시 SyntaxError: invalid syntax 구문 오류가 발생합니다.
    • 정식 배치 순서: 1. 위치 전용 변수 → 2. 슬래시(/) → 3. 일반 매개변수 → 4. 가변 위치 인자(*args) 또는 단독 별표(*) → 5. 키워드 전용 변수 → 6. 가변 키워드 인자(**kwargs) 순입니다.
  • 함수 주석 및 타입 힌트 (Type Hinting) 프로토콜
    • 파이썬은 동적 타이핑 언어이나, 코드의 명세성과 안정성을 위해 매개변수와 반환값의 타입을 힌팅하는 주석 문법을 공식 지원합니다.
    • 구문 규격: def plus(num1: int, num2: int) -> int: 형태로 매개변수 뒤에 콜론과 타입, 몸체 진입 전 화살표 기호(->)를 사용하여 반환 타입을 지정하여 명시합니다.
    • 함수를 인자로 갖는 타입 힌트: 파이썬은 함수를 일급 객체(First-Class Object)로 다루므로, 다른 함수의 매개변수 인자로 함수 자체를 전달할 수 있으며 주석 명세 구문 내에 상시 배치 작동이 가능합니다.

10장. 모듈화 및 패키지 인프라, 표준 turtle 그래픽스 제어

모듈 간의 상호 의존성을 낮추는 아키텍처 규칙과 독립 스크립트 실행 여부를 판별하는 내장 시스템 변수 통제, 그리고 터틀 물리 제어 API 리스트입니다.

  • 모듈화 프로그래밍 핵심 설계 이론
    • 모듈 (Module): 독립적인 네임스페이스를 보유한 단일 파이썬 코드 소스 파일(.py 파일 단위)입니다.
    • 패키지 (Package): 모듈 파일들을 물리적으로 그룹핑하기 위해 계층적으로 구성한 하위 폴더 시스템입니다.
    • 모듈화의 4대 가치 장점
      • 단순성(Simplicity): 당면한 거대한 전체 문제 대신 작고 명확한 모듈 단위의 당면 과제에만 집중 환경을 제공합니다.
      • 유지관리성(Maintainability): 모듈 간 상호 의존성을 최소화하도록 결합도를 낮추어 단일 모듈 수정이 타 프로그램 영역에 미치는 오염 충격을 완전 방지 및 방어합니다.
      • 재사용성(Reusability): 정형화된 모듈 인터페이스를 호출하여 다른 프로그램 코드 영역에서 쉽게 재사용함으로써 중복 복제 코딩 필요성을 근본적으로 소멸시킵니다.
      • 범위지정(Scoping): 모듈별로 격리된 독자 네임스페이스가 자동 정의되므로 여러 작업 영역 간의 식별자(변수명, 함수명) 이름 충돌 현상을 전면 회피합니다.
  • 내장 특수 변수 __name__ 을 사용한 독립 실행 진입점 통제 매커니즘
    • 개요 및 필요성: 파이썬 파일은 단독으로 작동하는 '독립 스크립트 역할'과 다른 파일에 부품으로 불러오는 '모듈 역할'을 동시에 수행함. 문장 import 명령어를 통해 불러와질 때는 내부 테스트용 실행 코드들이 가동되지 않도록 철저히 격리 통제해야 오염을 방지할 수 있습니다.
    • 내장 특수 변수 __name__의 성질: 인터프리터가 내부적으로 실행되는 모듈의 이름을 자동 저장하는 시스템 고유 변수입니다.
    • 독립 실행 시 코드 현황: 개발자가 프롬프트 콘솔 창에서 python 소스파일.py 명령을 내려 파일 자체를 직접 메인으로 기동했을 경우, 내장 특수 변수 __name__ 공간에는 문자열 '__main__'이 자동 세팅되어 적재됩니다.
    • 외부 임포트(import) 시 코드 현황: 다른 실행용 스크립트 파일에 의해 부품 모듈로써 import 모듈이름 형태로 호출되었을 경우, 내장 특수 변수 __name__ 공간에는 문자열인 '모듈이름' 자체가 자동 세팅되어 적재됩니다.
    • 진입점 통제 제어 구문 수립: 모듈 하단부에 if __name__ == '__main__': 조건문을 설계 배치함. 이 파일이 직접 메인으로 가동되었을 때만 True가 되어 하단 테스트 로직이나 본 코드가 구동하고, 타 파일에 의해 임포트되었을 때는 False가 되어 무시되도록 완벽한 실행 제어 가드를 구축합니다.
  • 내장 그래픽스 turtle 시각화 제어 API 상세 명세서
    • t.shape(name): 거북이의 외관 개체 모양을 'classic', 'turtle', 'arrow', 'circle', 'square', 'triangle' 형태로 가시화하여 전환합니다.
    • t.setup(width, height): 드로잉 스크린 그래픽 윈도우 창의 가로 및 세로 길이를 픽셀(Pixel) 단위를 기준으로 실시간 강제 조정 및 빌드 생성합니다.
    • t.speed(speed_code): 속도는 1에서 10까지 빨라지며, 정수 0을 인자로 대입 시 애니메이션 딜레이가 완전히 삭제된 최고 속도(즉시 렌더링)로 연산을 수행합니다.
    • t.pencolor(color_str): 드로잉 선 꼬리 잉크의 색상을 'red', 'blue' 등 표준 색상 텍스트로 치환 변경합니다.
    • t.write('문자열'): 거북이가 위치한 스크린 현재의 절대 좌표 점 위 지점에 커스텀 텍스트 글자를 직접 인쇄하여 표출합니다.
    • t.goto(x, y): 현재 정렬 방향을 유지한 상태에서, 스크린 가상 정중앙인 (0, 0) 원점 기준 명시된 2차원 공간의 절대 좌표 (x, y) 지점을 향해 다이렉트로 일직선 강제 이동을 제어합니다.
    • t.home(): 거북이 개체를 스크린 정중앙 위치 좌표 (0,0)인 초기 홈 위치로 복귀시키고 바라보는 각도와 시선도 초기화합니다.
    • t.clear(): 터틀 스크린 윈도우 창 내부에 그려진 모든 그래픽 선 데이터를 일괄 소멸 파괴하여 깨끗한 백지 상태로 초기화합니다.

11장. 서드파티 NumPy 행렬 연산, Matplotlib 데이터 시각화 및 최신 match-case 구조

실무 엔지니어링의 정수인 고성능 외산 패키지 구동 파이프라인과 최신 구조적 분기 문법의 세부 명세입니다.

  • 서드 파티 패키지 유통 및 설치 파이프라인 (PIP)
    • PyPI (Python Package Index): 전 세계 개발자가 검증한 서드파티 라이브러리를 공식 집약 및 보존해 둔 웹 색인 서버입니다.
    • pip.exe 진입 규칙: 운영체제(OS) 환경 내 다중 파이썬 버전 충돌 오염을 방지하기 위해, pip는 Python 패키지 설치 도구이며, 일반적으로 터미널 또는 명령 프롬프트에서 pip install 패키지명 형식으로 사용할 수 있습니다.
  • NumPy 고속 행렬 연산 아키텍처 레퍼런스
    • ndarray 자료형: 파이썬 일반 list와 달리 내부의 모든 데이터 원소가 동일한 C언어 기반 메모리 규격으로 연속 적재되어, 반복 루프 없이 수만 건의 행렬 원소 전체에 대한 초고속 일괄 벡터 사칙연산을 일괄 타격 가동합니다.
    • 행렬 곱 연산 API: 일반 곱셈 기호(*)는 동일 위치 원소끼리의 단순 곱만 수행함. 선형대수학 규격의 정밀 행렬 곱 연산을 연동 가동하려면 np.matmul(A, B) 또는 객체 내장형 메서드인 A.dot(B) 인터페이스를 호출 처리해야 합니다.
  • Matplotlib.pyplot 시각화 엔진 그래픽 조작 가이드
    • plt.figure(figsize=(a, b)): 출력될 전체 시각화 그림 및 차트 프레임의 가로 및 세로 물리 크기를 인치(Inch) 규격 단위로 사전 설정합니다.
    • plt.plot(data, color, linewidth, linestyle): 기본 꺾은선 그래프(Line Plot)를 드로잉 생성하며, color='blue', linewidth=3, linestyle=':'(점선 지정) 등의 세부 그래픽 속 인자를 키워드로 정밀 지정 제어가 가능합니다.
    • plt.bar() / plt.barh(): 세로형 표준 막대그래프 및 데이터 축이 좌우로 전개되는 수평 가로 막대그래프 렌더링을 확정합니다.
    • plt.scatter(x, y): 2차원 데이터 집합 축의 조밀 분산도를 가시화 표출하는 정밀 산점도(스캐터 플롯) 그래프를 생성합니다.
  • 구조적 패턴 매칭 (match-case) 구문 문법의 정밀 명세
    • 개요 및 차별성: 파이썬 최신 규격에 탑재된 다중 흐름 제어 구문으로, 기존 타 언어의 단순 switch-case 구조를 넘어서 유연한 데이터 시퀀스 매칭 및 변수 바인딩까지 완벽히 지원합니다.
    • 파이프 연산자 (|) 패턴 결합: case 400 | 404 | 405: 형태로 논리 OR 의미를 지닌 파이프 기호를 결합 기술하여 여러 다중 조건 매칭 패턴을 단 한 줄의 case 문 블록 아래로 수렴 통합 제어가 가능합니다.
    • 와일드카드 패턴 (case _:): 상단에 기술된 명시적 상수 패턴들과 일치하지 않고 남은 전체 예외 데이터 케이스를 오류 없이 안전하게 수집 흡수하여 프로그램 크래시 다운 현상을 방어하는 최하단 종단 가드 블록입니다.

12장. 3대 핵심 컴프리헨션 축약형 문법 및 내장 맵(map)·필터(filter) 구조

파이썬 특유의 고효율 한 줄 컬렉션 빌드 기술과 메모리를 극한으로 아끼는 지연 평가 연산 체계입니다.

  • 3대 컬렉션 컴프리헨션 (Comprehension) 축약 구조 레퍼런스
    • 개요 및 속도적 우위: 빈 컨테이너를 먼저 선언한 뒤 .append() 루프 구문을 반복 수행하는 구식 방식 대비, 파이썬 인터프리터 내부에서 C언어 기반 루프 패턴 최적화 연산이 다이렉트로 가동되어 실행 속도가 비약적으로 향상됩니다.
    • 리스트 컴프리헨션 명세: [expression for item in iterable if condition] → 조건식(if)이 참(True)으로 판별되는 요소 item만 선별하여 표현식(expression) 연산을 거친 최종 결과물 리스트로 즉시 리터럴로 반환합니다.
    • 딕셔너리 컴프리헨션 명세: {key_expr: value_expr for item in iterable if condition} → 반복 요소를 가공하여 단 한 줄 문장 서식으로 새로운 Key: Value 쌍을 만족하는 조립형 사전 객체를 단일 빌드하여 생성합니다.
    • 집합 컴프리헨션 명세: {expression for item in iterable if condition} → 중괄호 { } 기호 서식 내에 컴프리헨션을 구현하며, 생성 도중 중복된 원소 값이 발생하면 세트 집합 고유의 특성에 따라 즉시 자동 중복 제거 및 파괴 처리를 수행하며 오직 유일 원소만 보존합니다.
    • 제너레이터 컴프리헨션 (Generator Expr) 구조: 일반 소괄호 (expression for item in iterable) 서식으로 감싸 선언하며 모든 값을 메모리에 미리 올리지 않고 next 호출 시마다 늦게 평가(Lazy evaluation)하여 연산 메모리를 절약합니다.
  • 고성능 데이터 맵(Map) 및 필터(Filter) 연산 체계
    • map(func, iterable) 함수형 엔진: 반복 루프문을 소스에 직접 수립 코딩하지 않고, 타겟 컬렉션 내부의 모든 개별 원소 항목 데이터들을 첫 번째 인자로 매핑해 둔 매개변수 함수(func)의 인자로 실시간 순차 자동 투입하여 결과값을 일괄 변환 처리합니다.
    • filter(cond_func, iterable) 함수형 엔진: 컬렉션 내부의 원소들을 조건 판별용 함수에 대입하여, 그 결과가 오직 참(True)으로 평가 및 판단되는 원소 데이터들만 안전하게 골라내어 필터링하여 리턴합니다.
    • filter(None, iterable) 특수 매커니즘: 필터 함수 자리에 함수 이름 대신 파이썬 내장 객체 None을 인자로 지정해 주면, 컬렉션 내의 원소 중 거짓(False)을 유발하는 모든 원소(즉, 숫자 0, 실수 0.0, 빈 문자열 '', 빈 리스트 [], 빈 딕셔너리 {}, None, False 등)를 예외 없이 일괄 자동 필터링 파괴 제거하고 오직 참 값 원소만 압축하여 추출합니다.
    • 이터레이터 객체 반환 특성: map과 filter 함수의 결과물은 그 자체로 주소 포인터만 유지하는 이터레이터 객체 상태입니다. 따라서 최종적으로 데이터를 완전하게 눈으로 확인하고 슬라이싱 등에 연동 활용하려면 반드시 list(map(...)) 또는 list(filter(...)) 형태로 **리스트 명시적 캐스팅(Casting)**을 결합 설계해야만 정상 작동합니다.
  • 표준 패키지 내 특수 목적 컬렉션 인프라 명세서
    • collections.defaultdict (초기값 보장 사조함): 새로운 키를 조회하거나 사전에 삽입할 때, 설령 해당 키가 사전에 존재하지 않더라도 KeyError를 터뜨리지 않고 사전에 지정한 팩토리 함수(int, list 등)의 초기 기본값(0 또는 빈 리스트 [])으로 해당 키를 실시간 런타임에 자동 초기화 생성해 주는 특수 사전 모듈입니다.
    • operator 모듈 (함수형 연산자 패싱기): 더하기(+), 빼기(-), 미만(<) 등의 수학적 연산 기호 기법들을 그 자체로 타 함수의 인자나 변수 값으로 안전하게 전달할 수 있도록 함수형 규격 인터페이스(operator.add, operator.sub, operator.lt 등)로 완전 치환 지원해 주는 성능 지향형 표준 모듈입니다.
